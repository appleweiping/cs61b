== BucketsSpeedTest ==
# Insert N random Strings of length L=10 as <String,Integer> pairs.

N = 100000, L = 10
  MyHashMapALBuckets (ArrayList)    : 0.13 sec
  MyHashMapLLBuckets (LinkedList)   : 0.14 sec
  MyHashMapTSBuckets (TreeSet)      : 0.19 sec
  MyHashMapHSBuckets (HashSet)      : 0.18 sec
  MyHashMapPQBuckets (PriorityQueue): 0.13 sec

N = 500000, L = 10
  MyHashMapALBuckets (ArrayList)    : 0.50 sec
  MyHashMapLLBuckets (LinkedList)   : 0.54 sec
  MyHashMapTSBuckets (TreeSet)      : 0.68 sec
  MyHashMapHSBuckets (HashSet)      : 0.99 sec
  MyHashMapPQBuckets (PriorityQueue): 1.00 sec

# Takeaway: with a good hash + a bounded load factor, buckets stay short (O(1)
# expected length), so the simple list-backed buckets (AL/LL) are fastest.
# Tree/heap buckets pay an O(log b) per-bucket cost that only helps when many
# keys collide. Runtime scales ~linearly in N (amortized-O(1) put via resize).
